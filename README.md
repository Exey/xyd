### 2. О проекте
```markdown
## О проекте

Мы составляем рейтинг фотографов, работающих в жанре ню, по аналогии с индексом TIOBE.
Цель — создать прозрачную систему оценки профессионального уровня, основанную на:

- **10 метриках**, покрывающих технику, стиль, признание и этику.
- **Шкале от 0 до 10** с подробным описанием каждого уровня.
- **Открытых обсуждениях**: любой может предложить пересмотр оценки, приложив обоснования (ссылки, публикации, аргументы).

Исходный список и текущие оценки находятся в `SHORTLIST.md`.  
Веб-интерфейс (GitHub Pages) автоматически подтягивает эти данные и даёт возможность отправить заявку на пересмотр прямо в Issues.
```

### 3. Как устроен SHORTLIST.md
```markdown
## Формат данных

Список фотографов хранится в `SHORTLIST.md` в виде человекочитаемой таблицы с разделителем `;`.  
Столбцы:

| Поле | Описание |
|------|----------|
| Имя | Имя и фамилия (или псевдоним) |
| Персональный сайт | URL основного сайта (если есть) |
| Упоминания | Ссылки на профили / публикации / видео через пробел |
| Точки обновления | Каналы/аккаунты, по которым можно отслеживать новые работы (до двух) |
| Был пост в ноябре 25? | «Да» / «Нет» / пусто — маркер активности в указанный период |
| Оценка худсовета | Текущая числовая оценка (например, `6.5`) |
| (пустые колонки) | Зарезервированы для будущих полей |

**Пример строки:**
```
Илья Рашап;https://rashap.com;https://ru.wikipedia.org/wiki/Рашап,_Илья_Юлианович;;Нет;7;;;
```
```

### 4. 10 метрик оценки
```markdown
## Критерии оценки

Каждый фотограф оценивается по 10 метрикам (от 1 до 10 баллов). Итоговый балл — среднее взвешенное/консенсусное значение, отражающее сумму профессиональных качеств.

| # | Метрика | Что измеряет |
|---|---------|--------------|
| Ⓜ️1 | Узнаваемость авторского стиля | Насколько легко узнать работы без подписи |
| Ⓜ️2 | Техническое мастерство | Свет, композиция, ракурс, фокус |
| Ⓜ️3 | Работа с моделью | Эмоциональная выразительность, естественность поз |
| Ⓜ️4 | Сила и оригинальность концепции / идеи | Глубина и новизна художественного замысла |
| Ⓜ️5 | Внимание к деталям | Отсутствие визуального «мусора», аккуратность |
| Ⓜ️6 | Качество постобработки / цветокоррекции | Ретушь, колористика, техническая сложность |
| Ⓜ️7 | Наличие целостных серий / проектов | Умение создавать связные истории, а не только одиночные кадры |
| Ⓜ️8 | Публикации и медиа-присутствие | Статьи, интервью, публикации в прессе, соцсети |
| Ⓜ️9 | Выставочная деятельность и коллаборации | Выставки, работа с брендами, галерейные проекты |
| Ⓜ️10 | Репутация и этичность практик | Отзывы моделей, соблюдение профессиональной этики |
```

### 5. Шкала уровней (легенда)
```markdown
## Уровни рейтинга

Каждый балл — это **устойчивая характеристика** текущего этапа карьеры.  
Ниже — сводная легенда с примерами.

### 0 / 0.5 / 1.0 — Крайне низкий
Полное отсутствие технической базы, грубые ошибки, серьёзные этические нарушения.

### 2.0 — Начинающий с потенциалом
Слабый свет, шумы, нерезкость, но заметны зачатки стиля или интересной концепции.

### 3.0 — Любитель с амбициями
Базовые ошибки (завал горизонта, пересветы), попытки позиционировать себя как фотографа без понимания визуального языка.

### 4.0 — Пред-профи
Огрехи в деталях, неестественные позы, отсутствие целостных серий. Иногда удачный кадр, но не может повторить успех.

### 5.0 — Профессионал
Стабильная техника, но нет уникального стиля. Работы эклектичны, эмоционально нейтральны. Надёжный ремесленник.
- *Примеры:* Чуприн, Плотников, Стефанович

### 5.5 — Профессионал с чертами художника
Поиск своего голоса, первые сильные персональные проекты, целостные серии. Стиль ещё формируется.
- *Примеры:* Gene Oryx, Stakis Laus

### 6.0 — Локально признанный художник
Узнаваемый стиль, признание в профессиональной нише, стабильно высокий технический уровень.
- *Примеры:* Exey Panteleev, Сергей Ленин

### 7.0 — Высокий уровень
Яркий стиль, заметное медийное присутствие (мастер-классы, интервью), мощные серии.
- *Примеры:* Марат Сафин, Дима Борисов, Илья Рашап, Павел Киселёв, Чернядьев

### 7.5 — Художник на пороге международного признания
Прорыв за пределы страны: публикации в Kaltblut, Metal, I‑D, зарубежные выставки. Стиль приобретает глобальную ценность.
- *Примеры:* Влад Локтев, Олег Зотов

### 8.0 — Международно известный
Регулярные съёмки для Vogue, Harper's Bazaar, персональные выставки в топ-галереях. Виртуозная техника и инновационная постобработка.
- *Примеры:* Rankin, Рен Ханг

### 9.0 — Выдающийся
Сильнейший художественный почерк, новаторские концепции, влияние на индустрию.
- *Пример:* Дэвид Лашапель

### 10.0 — Легендарный
Работы — часть визуальной культуры, влияние на поколения, музеи, учебники.
- *Примеры:* Хельмут Ньютон, Ирвин Пенн, Ги Бурден
```

### 6. Как участвовать в оценке (через Issues)
```markdown
## Как предложить изменение рейтинга

Мы используем GitHub Issues как публичный худсовет.  
Вот два сценария:

### А) Изменить оценку существующему фотографу
1. Перейдите в [Issues](https://github.com/ваш-логин/ваш-репозиторий/issues/new/choose).
2. Выберите шаблон **«Пересмотр оценки»**.
3. Заполните:
   - Имя фотографа (как в SHORTLIST.md)
   - Предлагаемую новую оценку (0–10)
   - Обоснование: какие метрики изменились, приложите ссылки на новые работы, публикации, выставки.
4. Отправьте Issue. Начнётся обсуждение; окончательное решение принимают мейнтейнеры репозитория.

### Б) Добавить нового фотографа
1. Создайте Issue по шаблону **«Добавление фотографа»**.
2. Укажите имя, ссылки на портфолио, актуальные упоминания и предложите предварительную оценку.
3. После обсуждения и консенсуса запись будет добавлена в `SHORTLIST.md`.

### В) Быстрая ссылка с веб-страницы
На странице рейтинга у каждой строки есть кнопка **«Оспорить»** — она сформирует Issue с предзаполненным названием и ссылкой на фотографа.
```

### 7. Техническая часть (JS и GitHub Pages)
```markdown
## Как это работает технически

- **SHORTLIST.md** — единственный источник истины. Парсится JavaScript-скриптом на клиенте.
- **GitHub Pages** хостит статический сайт (`index.html`), который:
  - Читает `SHORTLIST.md` через `fetch`.
  - Строит сортируемую таблицу рейтинга.
  - Генерирует ссылки для создания Issue с предзаполненными параметрами.
- Никаких баз данных или серверной логики — всё прозрачно и хранится в репозитории.

Чтобы обновить данные, достаточно изменить `SHORTLIST.md` и сделать коммит — сайт подтянет изменения автоматически.
```
